# Security Policy

Thank you for helping improve the security of this project.

## Reporting a Vulnerability

If you believe you have found a security vulnerability, **please do not open a public GitHub issue**.

Instead, report it privately by contacting the maintainer:

- GitHub: https://github.com/ErichStyger
- Email: erich.styger@hslu.ch

Please include, if possible:

- affected repository and version
- target hardware and software version
- steps to reproduce
- impact of the vulnerability
- proof-of-concept or suggested fix (if available)

I will acknowledge receipt as time permits and investigate the report.

## Supported Versions

As this project consists of multiple repositories, examples, and reference implementations, only the latest revision on the `main` branch is actively maintained.

| Version | Supported |
|---------|:---------:|
| `main` | ✅ |
| Older releases | ❌ |

## Scope

This policy primarily applies to:

- source code in this repository
- build scripts and tooling
- documentation that could result in insecure use

The following are generally considered out of scope:

- vulnerabilities in third-party tools, SDKs, or libraries
- issues already fixed upstream
- hardware-specific limitations or errata

## Responsible Disclosure

Please allow reasonable time to investigate and prepare a fix before publicly disclosing a vulnerability.

Thank you for helping make embedded software more secure.
